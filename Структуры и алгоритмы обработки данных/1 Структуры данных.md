Программа для ЭВМ(компьютера) представляет собой формальное описание решения некоторой задачи реального мира. Когда человек употребляет термин "программа", то он подразумевает не только последовательность оператора некоторого языка программирования, но и набор различных информационных объектов, над которыми выполняют те или иные действия операторы программы. 
**Данные** - это любой набор знаков, рассматриваемый безотносительно к его содержательному смыслу. Данные обычно изображают некоторую информацию, которую можно получить, если известен смысл приписываемый данным.
В области обработки данных термины "информация" и "данные" часто используются в одинаковом значении.
Самое простое данное, с точки зрения организации - **бит**. А предела сложности не существует. Единственное ограничение - здравый смысл.
При программировании часто используют различные совокупности данных, организованные определенным образом. Организованные совокупности данных называют "**структурные данные**". В целом в программировании обработка структур данных занимает одно из главных мест, а из изучение относится к фундаментальным разделам информатики. Когда программист создает алгоритм для решения задачи, он также создает и структуру данных, которая обрабатывается этим алгоритмом. Выбор структуры данных может в значительной степени повлиять на объем требуемой для обработки памяти, а также на скорость обработки.
Программу, которая предназначена для решения некоторой задачи, можно рассматривать, как результат объединения структур данных и алгоритма. Для решения одной и той же задачи может существовать ряд различных программ. На это влияет:
1. Способ представления и упорядочивания данных
2. Постановка задач
## Преобразование указателя типа void
В отличии от всех других базовых типов у типа данных void нет ни набора допустимых значений, ни набора допустимых операций. Поэтому нет смысла и возможности описания переменных относящихся к типу void. Поскольку нет значений, которые им можно присвоить. Этот тип данных введен в язык C для выполнения специальных функций:
1. Для указания того, что функция не возвращает никаких значений
2. Для указания того, что функции не передаются никакие аргументы
3. Для объявления указателя на тип данных void (Объявление указателя на тип void означает, что этот указатель может указывать на любой тип данных)

Обычные указатели всегда указывают на какой-то определенный объект программы и только заранее определенный тип данных. Существует правило: величина типа void* может быть преобразована к величине любого другого указательного типа (прописать перед указателем преобразование типа `(тип*) имя_указателя_void = адрес какой-то переменной`)
Указатель * z указывает не на адрес какого-то конкретного объекта программы, а просто на адрес памяти, по которому может храниться любой объект некоторого типа. Поэтому указатель void* иногда называют адресом указателя. Часто функции в качестве своего значения возвращают указатель типа void.
Пусть имеется некоторая функция f, она что-то делает и возвращает адрес, то есть указатель типа void. Возвращающая функция берет содержимое по этому адресу и интерпретирует его по своему усмотрению
## Понятие элемента структуры данных.  
Структура данных - это организованная структура данных или организованный набор данных. Любая структура данных состоит из элементов данных, все элементы данных одной структуры связаны между собой одной или несколькими связями. Все связи одного элемента данных с другими элементами данных образуют элементы отношений рассматриваемого элемента данных. Элемент отношений состоит из конечного множества пар, каждое из которых отражает одну из связей элементов с другими элементами данных.
Пара содержащая элемент данных и связанный с ним элемент-отношений называется элементом структуры.

Структура данных - это совокупность связанных данных и множество правил, определяющих их организацию и способ доступа к элементам данных. Множество правил = множество отношений.
С точки зрения решения реальной задачи элемент данных представляет собой наименьшую логически неделимую структуру данных. Смысловая неделимость элементов связана с тем, что именно элементы являются отражением отдельных свойств отдельно взятых характеристик тех или иных объектов, процессов или явлений, имеющихся в реальной задаче. Элемент данных, взятый обособленно от других элементов не может быть носителем полной информации об объекте. Однако элементы данных можно определить безотносительно какой-либо структуры данных. Один и тот же элемент данных может входить в различные простые или более сложные структуры данных, относящиеся к разным по своему характеру объектам материального мира.
## Изображение структуры данных
Для наглядного изображения структуры данных часто используют ее графическое представление. Оно должно отражать элементы-данных структуры и связи, то есть отношения между элементами
Пример 1. Использование неориентированного графа
Имеется некоторая карта, на которой обозначены города (A, B, C, D, E ). Нужно представить эту карту для ее обработки с помощью компьютера. Нам требуется составить структуру данных, в которой мы будем хранить всю информацию об этих городах. 
Элементы-данных нашей структуры - города. Они должны хранить информацию о городе (название, площадь, население, некоторые точки города, и тд.)
Элементы-отношений - наличие дорог от одного города до другого
Пример 2. Ориентированный граф
Города соединены с следующем порядке.
A -> B, C
B -> A, D, E
C -> D
D -> B, E
E -> C
Нужна программа, которая определяет самый короткий путь из одного города в другой

В некоторых случаях отношения между элементами-данных в структуре настолько простые и очевидные, что не возникает особой необходимости их отражения. То есть элементы-отношений могут в явном виде отсутствовать.
Часто изображается не вся структура, а лишь один или несколько ее элементов. Для удобства элемент разбивается на фиксированные поля, каждое из которых содержит простые данные, элемент-данных или элемент-отношений. Очень часто элемент-отношений сводится к указателю.

Все, что мы рассматривали до этого, рассматривалось без учета представления структуры в машинной памяти. Такая структура данных называется абстрактной или логической, но компьютер может обрабатывать только такие структуры данных, которые тем или иным способом представлены в машинной памяти. Структура данных, представленная в машинной памяти называется физической структурой или структурой хранения, внутренней структурой, структурой памяти.
Когда программист конструирует структуры данных он должен различать понятие логической и физической структур. Логическая структур данных это идеальная схема представления или модель соответствующего данного. Физическая структура данных это соответствующая схема размещения или хранения данных в памяти компьютера, то есть она отражает способ физического представления данных

Представим, что у нас есть некоторая задача, которую нам необходимо решить. Для этой задачи определен некоторая предметная область, в рамках которой решается задача. Для решения задачи мы должны формализовать предметную область. То есть отобразить реальный мир в виде некоторых конкретных и абстрактных понятий между которыми существуют определенные связи. Для этого мы выявляем только те понятия, которые нам интересны и нужны с точки зрения решения поставленной задачи, при этом все ненужное мы отбрасываем.
Для построения корректной модели данных необходимо выявить только те понятия, которые интересны и нужны с точки зрения решения задачи и отбросить все ненужное. При этом необходимо:
1. Дать свое имя каждому понятию
2. Установить связи между именами-понятиями

Такой процесс выявления требуемых для решения задачи данных называется **проектированием данных**. Результатом проектирования является модель данных или другими словами структура данных. Здесь еще не были учтены конкретные условия реализации. Для их учета существует 2 пути:
1. Когда мы конкретные условия реализации будем учитывать непосредственно в самой структуре данных. В таком случае при изменении условий реализации будет меняться сама структура данных. Соответственно может существенно измениться алгоритм. Возможно придется переделывать всю программу
2. Когда структура данных отделяется от описания конкретных условий реализации. В этом случае структура данных представляется в виде 2 структур данных. 1 - структура без учета среды реализации. 2 - структура с учетом среды реализации

В настоящее время при решении задач придерживаются именно 2-ого пути решения. В таком случае проектирование структуры данных осуществляется в 2 этапа.
1. Логическое проектирование структуры данных
2. Физическое проектирование структуры данных

Термины логические и физические используются для описания различных аспектов данных. Логический указывает на то, как данные представляются пользователю или программисту. Используются термины: логическое отношение, логическая структура данных, логическое описание данных. Термин физический указывает каким образом данные хранятся в среде хранения и в этом случае используют следующие выражения: физическое отношение, физическая структура данных, физическое описание данных.

При логическом проектировании структуры данных не учитываются физические характеристики компьютера, особенности программного и математического обеспечений. Мы можем даже не знать где и с помощью каких средств мы будем решать нашу задачу. Все это учитывается только на стадии физического проектирования структуры данных. В общем случае логическая и соответствующая ее физическая структуры данных не совпадают. Степень их различия определяется самой структурой и особенностями среды реализации.
В случае работы с фалами и двумерными массивами переход от логической к физической структуре данных происходит автоматически с помощью уже имеющегося ПО. Как файл, так и массив являются встроенными структурами данных для любого языка и для них уже предусмотрен механизм перехода от логической структуры данных к физической и наоборот. В общем случае для логической структуры, которая не является встроенной, потребуется самостоятельно разработать ПО для перехода к физической структуре данных и наоборот. В следствии различия логической и соответствующей физической структур должно существовать ПО, осуществляющее отображение логической структуры в физическую и наоборот. Это ПО должно обеспечивать доступ к физическим структурам и выполнение над ними различных операций, когда мы выполняем операции на логической структурой. 
## Логические структуры данных
Нельзя перечислить все логические структуры данных. Они определяются множеством решаемых задач, однако опыт разработки ПО показал, что существуют такие структуры данных, которые большинство специалистов считают основными или базовыми. К ним можно свести данные при решении большинства задач. 
К основным (базовым) структурам данных относятся: вектор, массив, строка, стек, дек, очередь, последовательность, множества, запись, таблица, связные списки (односвязные, двусвязные, кольцевой односвязный, кольцевой двусвязный, многосвязный), древовидные структуры, графовые структуры общего вида
## Типы данных и структуры данных
Для любого языка программирования имеется набор типов данных, которые на нем можно обрабатывать. Для программиста это означает следующее: он может оперировать только теми данными, которые относятся к допустимому для языка программирования типу и соответственно составленную логическую структуру данных он может представить на языке программирования только с использованием типов данных, существующих в этом языке. Часть самых простых структур, с которыми может работать программа, встроили в язык программирования и называли типами данных. Это значит что процесс отображения и сама структура данных формируется автоматически. Программист может не знать как и не утруждать себя знаниями о том что такое логическая структура, что из себя представляет логическая структура и как происходит отображение одного в другое. При разработке языков программирования к ним предъявляется рад требований. В том числе и к их типам данных. Для нас интересно 2 момента:
1. Описание объектов реального мира с использованием средств языка программирования должно быть простым для понимания и небольшим по объему
2. Реализация языка программирования должна обеспечивать высокую эффективность работы существующих компьютеров

Допустим все известные структуры данных включили в язык. Так мы удовлетворим первое требование, но эффективность работы всех программ резко снизится, так как в ней будет много неиспользуемого. Потому что не в любой программе одновременно требуются все структуры данных. Наоборот же, если язык будет обладать только минимальным набором структур данных, то возникнут трудности при описании предметных задач. Поэтому язык программирования в качестве встроенных типов данных включает те структуры, которые требуются для решения наибольшего числа задач того класса, для которых разрабатывается язык
## Классификация логических структур данных
Любая классификация определяется признаком, который положен в основу классификации. Можно выделить несколько наиболее важных признаков для классификации структур данных:
1. Отсутствие или наличие явно заданных связей между элементами структуры (несвязные структуры: вектор, строка, стек, дек, очередь, запись, таблица, последовательность или связные структуры: любые связные списки, древовидные структуры, графовые структуры общего вида)
2. Характер упорядоченности элементов структуры данных (линейные: все элементы структуры расположены на одном уровне, нелинейные: когда элементы структуры расположены на разных уровнях). В свою очередь линейные структуры данных делятся по характеру взаимного расположения элементов (структуры с последовательным расположением: строка, массив, стек, дек, очередь, последовательность. И структуры с произвольным связным расположением: односвязный простой список, двусвязный простой список, кольцевые одно- и двусвязные списки). Нелинейные: многосвязные списки, древовидные структуры и графовые структуры общего вида
3. По количеству элементов и\или количеству связей в самой структуре данных (само значение элемента-данных может меняться у любой структуры). Здесь структуры делятся на 3 типа: статические (вектор, массив, запись, таблица), полу-статические (стек, дек, очередь) и динамические (все списковые структуры, древовидные структуры, графовые структуры, динамический вектор и динамическая матрица)
