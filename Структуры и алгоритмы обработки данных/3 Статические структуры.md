**Вектор** (одномерный массив) - одна из простейших структур данных, которая имеется во всех алгоритмических языка. Элементы вектор находятся друг с другом в единственно возможном отношении, а именно отношении непосредственного следования. Строгая упорядоченность элементов вектора позволяет пронумеровать их последовательными целыми числами, которые носят название "индексы элемента".
Логическая структура вектора полностью определяется количеством и типом элементов вектора. При описании логической структуры вектора ему обычно присваивается имя, указывается минимальное значение индекса, максимальное значение элементов и тип для каждого элемента вектора

**Массив** - с точки зрения общего подхода это вектор, каждый элемент которого в свою очередь так же является вектором. Формальное определение: к-мерный массив (k=1-...) это конечное упорядоченное множество (k-1)-мерных массивов, все элементы которых принадлежат одному и тому же типу.
Логическая структура массива задается следующим образом - обязательно задается имя, минимальное значение индекса по каждому измерению, тип элементов массива
Выделяют 2 частных случая для двумерных массивов:
1. симметричная квадратная матрица (у нее все элементы, расположенные симметрично относительно главной диагонали, попарно равны друг-другу). Такие массивы так же называют треугольными. Доступ к ним организуется таким образом, чтобы можно было обращаться к любому элементу исходной логической структуры. В том числе и к элементам, которые не представлены в памяти, на основе значений симметричных им элементам
2. разреженные массивы - это массив, большинство элементов которого равны 0. В памяти можно хранить лишь небольшое количество ненулевых значений элементов массива

**Запись** - конечное упорядоченное множество элементов различных типов данных. Элементы записи называют полями. Доступ к любому элементу записи осуществляется посредством имени, которое должно задаваться для каждого элемента на этапе описания логической структуры записи. Запись это обобщение понятия вектора, при котором не требуется однотипность элементов. Это простая запись. Для нее характерно, что каждый ее элемент является простым данным или цепочкой данных одного и того же типа. С точки зрения языка C запись это структура (`struct`). Общим видом записи является такая запись, в которой каждый ее элемент может быть в свою очередь записью более низкого уровня. При этом каждый элемент записи более низкого уровня так же может быть записью следующего уровня. Таким образом такая запись может содержать произвольное количество уровней и ее называют многоуровневой записью. Независимо от числа уровней в структуре записи, полезная информация содержится только в тех элементах, которые соответствуют листьям дерева, представляющего запись. Остальные элементы записи а также и имя записи используются для доступа к полезной информации записи. Элементы записи с полезной информацией называются содержательными элементами и в общем случае они располагаются на разных уровнях в логической структуре записи

**Таблица** - конечное число записей, имеющих одну и ту же организацию. Каждая запись входящая в таблицу называется элементом таблицы. Часто используют такую форму таблицы, в которой элемент представляет собой одноуровневую запись. При этом логическая структура таблицы изображается в виде последовательности расположенных друг под другом строчек одинаковой длинны представляющих элементы таблицы и разделенных на графы. Каждая графа соответствует определенному полю элемента таблицы. Обычно одно из полей всех элементов таблицы отводится для хранения ключа, являющегося уникальным для каждого элемента и используемого при доступе к соответствующему элементу. Доступ к каждому элементу таблицы осуществляется по ключу и целью доступа служит не отдельное данное, а целая запись. Типичные операции над таблицей - включение в нее данных о новом элементе, поиск заданного элемента с целью чтения его данных, исключение заданного элемента из таблицы. 

Рассмотренные структуры (вектор, массив, запись, таблица) характеризуются следующими свойствами:
1. постоянство структуры в течении всего времени ее существования
2. смежность элементов
3. простота и постоянство отношений между элементами структуры

Все это является свойствами статических структур.
